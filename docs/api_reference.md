# 🔌 API Reference: CDIE v4 FastAPI Backend

CDIE v4 (Causal Decision Intelligence Engine) exposes a high-performance REST API built with FastAPI. This document provides a detailed reference of the available endpoints for both **Online Inference** and **System Management**.

---

## 🏗️ 1. Online Inference (Causal Queries)

### `POST /query`
Process a natural language or structured causal query against the pre-computed **Safety Map**.

- **Payload**:
  ```json
  {
    "query": "What happens if SIM Box Fraud attempts increase by 20%?"
  }
  ```

- **Response Highlights**:
  - `point_estimate`: The estimated Average Treatment Effect (ATE).
  - `ci_lower` / `ci_upper`: 95% Confidence Intervals (Conformal Prediction).
  - `refutation_status`: Results of the DoWhy refutation tests (Placebo, Confounder, Subset).
  - `explanation`: Markdown intelligence report generated by the OPEA LLM.
  - `historical_analogies`: Relevant playbooks retrieved via TEI RAG.

---

### `POST /prescribe`
Find the top interventions to maximize or minimize a specific target (e.g., ARPU or Revenue Leakage).

- **Payload**:
  ```json
  {
    "target": "RevenueLeakageVolume",
    "maximize": false,
    "limit": 3
  }
  ```

- **Response**:
  Returns a prioritized list of interventions (e.g., "Tighten Fraud Policy by 15%") ranked by their estimated causal impact on the target.

---

## 🏗️ 2. System Status & Hardare

### `GET /health`
Returns the system health status, including Safety Map integrity hash and memory usage.

- **Response**:
  ```json
  {
    "status": "healthy",
    "safety_map_hash": "SHA-256...",
    "last_computed": "2024-03-24T12:00:00Z",
    "memory_mb": 142.5
  }
  ```

---

### `GET /benchmark/hardware`
Detects Intel CPU features (AMX, AVX-512) and checks if hardware optimizations are active.

---

### `GET /info`
Provides a detailed system overview, including the status of all 3 OPEA microservices (TextGen, Embedding, Reranking).

---

## 🏗️ 3. Causal Data & Visuals

### `GET /graph`
Returns the discovered Causal DAG in a JSON format compatible with React Flow, including edge validation status and causal weights.

---

### `GET /catl`
Returns the results of the **Causal Assumption Transparency Layer** tests (Faithfulness, Sufficiency, etc.) for the current model.

---

### `GET /temporal`
Returns time-lagged causal results from the PCMCI+ temporal analysis.

---

## 🏗️ 4. Expert Corrections (HITL)

### `POST /expert/correct`
Allows a domain expert to explicitly reject an edge in the causal graph.

- **Payload**:
  ```json
  {
    "from_node": "NetworkLoad",
    "to_node": "SIMBoxFraudAttempts",
    "action": "reject"
  }
  ```

- **Effect**:
  The correction is persisted to `data/prior_corrections.json` and will be applied as a hard constraint during the next run of the **Offline Pipeline**.
